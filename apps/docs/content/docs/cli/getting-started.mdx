This guide walks you through installing the CLI, authenticating, and initializing
your first project.

## Prerequisites

You need:

- Node.js 20 or later
- A Supabase account (create one at [supabase.com](https://supabase.com))

## Install the CLI

Install globally with npm:

```bash
npm install -g @supabase/cli
```

Or with pnpm:

```bash
pnpm add -g @supabase/cli
```

Verify the installation:

```bash
supa --version
```

## Authenticate

Log in to your Supabase account:

```bash
supa login
```

This opens your browser to authenticate. After logging in, enter the verification
code shown in your terminal.

For CI/CD environments or scripts, use a token directly:

```bash
supa login --token sbp_xxxxx
```

Get your access token from the
[Supabase dashboard](https://supabase.com/dashboard/account/tokens).

## Initialize your project

Run the init wizard in your project directory:

```bash
supa init
```

The wizard guides you through:

1. Selecting your organization
2. Selecting an existing project or creating a new one
3. Choosing a workflow profile (solo, staged, or preview)
4. Configuring schema management

After initialization, the CLI creates a `supabase/` directory with your project
configuration and displays your API credentials.

### Non-interactive initialization

For scripts or CI/CD, use flags to skip prompts:

```bash
# Link to existing project
supa init --project abc123xyz

# Create new project
supa init --org my-org --name my-project --region us-east-1
```

## Project structure

After initialization, your project looks like this:

```
your-project/
├── supabase/
│   ├── config.json      # Project configuration
│   ├── schema/          # SQL schema files
│   ├── migrations/      # Version-controlled migrations
│   ├── functions/       # Edge functions
│   └── types/           # Generated TypeScript types
└── .env                 # Environment variables (if new project created)
```

## Start developing

Start the dev server to watch for changes:

```bash
supa dev
```

This watches your schema and config files, automatically syncing changes to your
project.

Create a schema file:

```sql title="supabase/schema/public/posts.sql"
CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

The CLI detects the change and syncs it to your project.

## Push and pull

Pull the current state from your project:

```bash
supa project pull
```

Push local changes to your project:

```bash
supa project push
```

Preview changes without applying them:

```bash
supa project push --plan
```
