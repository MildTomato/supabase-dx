title: "supa project"
description: "Project operations"
---

Commands for working with the linked Supabase project.

These commands operate on the project linked in the current directory via `supa init`.

## Usage

```bash
supa project
```

## Subcommands

| Command | Description |
|---------|-------------|
| [`pull`](/docs/cli/reference/project-pull) | Pull remote state to local (remote → local) |
| [`push`](/docs/cli/reference/project-push) | Push local changes to remote (local → remote) |
| [`dev`](/docs/cli/reference/project-dev) | Watcher that auto syncs changes to hosted environment [long-running] |
| [`seed`](/docs/cli/reference/project-seed) | Run seed files against the database |
| [`seed-status`](/docs/cli/reference/project-seed-status) | Show seed configuration and files |
| [`api-keys`](/docs/cli/reference/project-api-keys) | List API keys for the project |
| [`profile`](/docs/cli/reference/project-profile) | View or change workflow profile |

## Examples

### Pull remote state

```bash
supa project pull
```

### Push local changes

```bash
supa project push
```

### Start development watcher

```bash
supa project dev
```
