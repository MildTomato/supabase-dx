This reference documents all available configuration options in `config.json`.

## Schema

The configuration file uses JSON Schema for validation. Add the schema URL
to enable editor support:

```json
{
  "$schema": "https://supabase.com/schema/config.json"
}
```

## Environment variables

Use the `env()` syntax to reference environment variables in your config:

```json
{
  "auth": {
    "external": {
      "google": {
        "client_id": "env(GOOGLE_CLIENT_ID)",
        "secret": "env(GOOGLE_CLIENT_SECRET)"
      }
    }
  }
}
```

The CLI loads environment variables from these files (in order of precedence):

1. Shell environment
2. `.env.local`
3. `supabase/.env`
4. `.env`

## Type definitions

TypeScript types are available for the configuration schema:

```typescript
import type { ProjectConfig } from '@supabase/cli';

const config: ProjectConfig = {
  project_id: 'my-project',
  workflow_profile: 'solo',
  api: {
    max_rows: 1000
  }
};
```
