---
title: "supa project dev"
description: "Watcher that auto syncs changes to hosted environment [long-running]"
---

## Usage

```bash
supa project dev [options]
```

## Options

| Option | Type | Description |
|--------|------|-------------|
| `-p, --profile <NAME>` | string | Profile to use |
| `--debounce <MS>` | string | Debounce interval for file changes (e.g., 500ms, 1s) |
| `--types-interval <INTERVAL>` | string | Interval for regenerating types (e.g., 30s, 1m) |
| `--no-branch-watch` | boolean | Disable git branch watching |
| `--seed` | boolean | Run seed files after schema sync (also re-seeds on schema changes) |
| `--no-seed` | boolean | Disable seeding even if enabled in config |
| `--dry-run` | boolean | Show what would be synced without applying |
| `-v, --verbose` | boolean | Show detailed pg-delta logging |
| `--json` | boolean | Output as JSON (events as newline-delimited JSON) |

## Examples

### Start development watcher

```bash
supa dev
```

### Watch with specific profile

```bash
supa dev --profile staging
```

### Preview changes without applying

```bash
supa dev --dry-run
```

### Watch with automatic seeding

```bash
supa dev --seed
```
